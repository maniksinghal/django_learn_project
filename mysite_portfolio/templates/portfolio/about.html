This is an about page (the actual implementation is in /portfolio)

We start by creating a model (database-schema) in models.py of the app
All the fields in the model are of type models.DataType
When working with image fields, following additional is required:
- pip3 install pillow
- Create a MEDIA_ROOT path in settings.py (generally top level media folder in project)
- Create any MEDIA_URL (just like URL path in urls.py) in settings.py
- Tie MEDIA_ROOT with MEDIA_URL in urls.py

Create migration for the new model: python3 manage.py makemigrations
- Check out project/app/migrations/__init__.py (auto-generated)
Register migration with the admin: python3 manage.py migrate

Adding entries in the model using admin

- Register the model with admin through admin.py: admin.site.register(Project) - Create superuser: python3 manage.py createsuperuser
- Open localhost:/admin (points to admin.sites.url)
  - Add entries graphically into the database.

Pulling entries into html

- projs = Project.objects.all()
- for proj in projs: proj.title, proj.description, ...
- Some custom code language support in Django to iterate, print etc.
- check mysite_portfolio/templates/portfolio/home.html for details
